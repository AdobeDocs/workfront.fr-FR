## Améliorations pour l’ensemble des utilisateurs et utilisatrices

Certaines fonctionnalités ont été publiées avant la version 2019.4. Pour connaître les versions antérieures à la version 2019.4, voir [Autres améliorations de la version 2019.4](../../../../product-announcements/product-releases/quarterly-release-archive/2019.4-release-activity/2019-4-other-enhancements.md).

La version 2019.4 contient les améliorations suivantes :

* [Améliorations des listes](#list-enhancements)
* [Améliorations mobiles](#mobile-enhancements)
* [Amélioration de la planification des ressources](#resource-scheduling-enhancements)
* [Autres améliorations](#other-enhancements)

### Améliorations des listes {#list-enhancements}

>[!IMPORTANT]
>

Ces améliorations seront ajoutées à nouveau à l’environnement de prévisualisation après la version de production 2019.4, et seront disponibles sur l’environnement de production dans une version ultérieure.

Voici le calendrier de suppression de ces options d’authentification unique :

* Prise en charge limitée : août 2018 - janvier 2019

  Les problèmes majeurs soumis pendant cette période seront résolus. Les problèmes de faible gravité ne seront pas traités.

* Abandon : janvier 2019 à novembre 2019 (avec la version 2019.4)

  Non pris en charge.

* Fin de disponibilité dans Workfront pour les nouvelles configurations : novembre 2019
* Migration obligatoire pour les clientes et clients existants : début 2020

### La prise en charge de TLS 1.0 et 1.1 est terminée. {#tls-1-0-and-1-1-support-is-ending}

Afin de fournir une sécurité optimale, Workfront exige que toutes les intégrations et le trafic web utilisant TLS 1.1 ou une version antérieure soient mis à jour pour utiliser TLS 1.2. Dans l’environnement de prévisualisation, TLS 1.0 est déjà désactivé.
