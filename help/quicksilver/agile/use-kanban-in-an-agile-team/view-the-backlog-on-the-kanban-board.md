# Ajoutez le journal en souffrance à la carte [!UICONTROL Kanban]

Vous pouvez afficher une colonne [!UICONTROL Backlog] directement sur votre panorama Kanban. La colonne [!UICONTROL Backlog] du panorama [!DNL Kanban] contient les 20 premiers éléments de votre journal sur le panorama [!DNL Kanban]. Les tâches et les problèmes doivent appartenir à un projet dont l’état équivaut à [!UICONTROL Actuel] pour apparaître dans le journal des travaux en souffrance.

## Ajoutez le journal en souffrance à la carte [!UICONTROL Kanban]

Par défaut, la colonne [!UICONTROL Backlog] est masquée. Pour afficher le journal en souffrance :

1. Cliquez sur l’icône *[!UICONTROL *Menu principal]** ![](assets/main-menu-icon.png) dans le coin supérieur droit de [!DNL Adobe Workfront], puis cliquez sur **[!UICONTROL Equipes]**.

1. (Facultatif) Cliquez sur l’icône **[!UICONTROL Changer d’équipe]** ![Icône Changer d’équipe](assets/switch-team-icon.png), puis sélectionnez une nouvelle équipe [!UICONTROL Kanban] dans le menu déroulant ou recherchez une équipe dans la barre de recherche.

1. Accédez au panorama Kanban sur lequel vous souhaitez afficher le journal en souffrance.
1. Cliquez sur **[!UICONTROL Afficher le Backlog]**.

>[!NOTE]
>
>Si vous ajoutez plusieurs équipes à un élément de journal, la tâche ou le problème s’affiche uniquement sur le journal de l’équipe principale. L’équipe principale est la première équipe affectée.

Pour plus d’informations sur le journal des travaux, voir [Gestion du journal des travaux agiles](../../agile/work-in-an-agile-environment/manage-the-agile-backlog.md).
