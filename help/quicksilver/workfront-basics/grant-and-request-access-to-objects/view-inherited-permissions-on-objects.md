# Afficher les autorisations héritées sur les objets

Votre administrateur Adobe Workfront peut vous accorder l’accès à l’affichage ou à la modification d’objets lors de l’attribution de votre niveau d’accès. Pour plus d’informations, voir [Accorder l’accès aux projets](../../administration-and-setup/add-users/configure-and-grant-access/grant-access-projects.md).

Outre le niveau d’accès que les utilisateurs reçoivent, vous pouvez leur accorder des autorisations sur des objets spécifiques que vous avez accès au partage. Pour plus d’informations sur les niveaux d’accès et les autorisations, voir [Comment les niveaux d’accès et les autorisations fonctionnent ensemble](../../administration-and-setup/add-users/access-levels-and-object-permissions/how-access-levels-permissions-work-together.md).

Les objets héritent des autorisations des objets parents dans Adobe Workfront.

Pour plus d’informations sur la hiérarchie des objets dans Workfront, voir [Comprendre les objets dans Adobe Workfront](../../workfront-basics/navigate-workfront/workfront-navigation/understand-objects.md).

## Affichage des autorisations héritées pour un objet

L’affichage des autorisations héritées est identique pour tous les objets.

Pour afficher les autorisations héritées pour un projet :

1. Accédez à un projet dont vous souhaitez afficher les autorisations de partage.
1. Cliquez sur le menu **Plus** ![](assets/more-icon.png), puis sur **Partage**.

1. Développez la liste **Autorisations héritées**.

   Cette liste répertorie les noms des utilisateurs qui ont accès au portefeuille ou au programme auquel appartient le projet et qui disposent également d’autorisations sur le projet.

   ![](assets/remove-inherited-permissions-on-project-nwe-350x475.png)

1. (Facultatif) Pour supprimer les autorisations héritées d’un objet, voir [Suppression des autorisations des objets](../../workfront-basics/grant-and-request-access-to-objects/remove-permissions-from-objects.md).

   >[!NOTE]
   >
   >Pour supprimer les autorisations héritées, vous devez disposer des autorisations Manage (Gérer) sur un objet.
