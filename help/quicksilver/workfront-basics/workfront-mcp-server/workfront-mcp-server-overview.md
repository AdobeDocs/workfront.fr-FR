# Présentation du serveur MCP Adobe Workfront

Le serveur MCP (Model Context Protocol) [!DNL Adobe Workfront] vous permet de travailler avec vos données Workfront par le biais d’une conversation en langage naturel, sur une plateforme agentique d’IA prise en charge. Au lieu de cliquer sur Workfront pour rechercher un projet, mettre à jour une tâche ou vérifier une approbation, vous demandez à la plateforme agentic d’IA de vous répondre en langage clair, et cette dernière effectue le travail à votre place.

Vous n’avez pas besoin de connaître l’API Workfront ou quoi que ce soit d’autre sur MCP pour utiliser le serveur MCP Workfront. Configurez la plateforme agentique d’IA pour parler à Workfront une fois, puis commencez à demander.

## Configuration et utilisation du serveur Workfront MCP

Avant de pouvoir utiliser le serveur MCP Workfront, un administrateur Workfront doit activer l’accès au serveur MCP dans Workfront et vous devez connecter votre plateforme d’agence AI à Workfront. Pour plus d’informations, voir [Configuration du serveur Adobe Workfront MCP](/help/quicksilver/workfront-basics/workfront-mcp-server/configure-workfront-mcp-server.md).

Après la configuration, vous pouvez demander à la plateforme IA agentic de rechercher, créer, mettre à jour et gérer les éléments Workfront. Pour plus d’informations sur les actions disponibles, les requêtes d’exemple et la gestion des données, consultez [Utilisation du serveur Adobe Workfront MCP](/help/quicksilver/workfront-basics/workfront-mcp-server/use-workfront-mcp-server.md).
