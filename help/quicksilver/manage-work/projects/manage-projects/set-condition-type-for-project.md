# Définir le type de condition d’un projet

En tant que chef de projet, vous pouvez déterminer comment la condition d’un projet est calculée en mettant à jour le type de condition du projet. La condition du projet est une représentation visuelle de l’avancement du projet.

## Définition du type de condition pour un projet

1. Accédez au projet pour lequel vous souhaitez mettre à jour le type de condition.
1. Effectuez l’une des opérations suivantes : 

   * Cliquez sur le menu **Plus** ![](assets/qs-more-menu.png) situé à droite du nom du projet, puis cliquez sur **Modifier**.
   * Cliquez sur **Détails du projet** dans le panneau de gauche.

   ![](assets/update-condition-type-nwe-350x108.png)

1. Dans le champ **Type de condition** , sélectionnez l’une des options suivantes :

   * **Manuel :** Le propriétaire du projet définit manuellement la condition sur le projet.

     Dans ce cas, le propriétaire du projet peut mettre à jour la condition du projet dans l’en-tête du projet ou dans la section Détails du projet .

   * **État de progression :** Workfront définit la condition en fonction de l’état de progression du projet. Pour plus d’informations sur le mode de calcul de l’état de progression, consultez la [présentation de l’état de progression du projet](../../../manage-work/projects/planning-a-project/project-progress-status.md).

1. Cliquez sur **Enregistrer** lorsque vous modifiez le type de condition dans la zone Modifier le projet.

   Cliquez sur **Enregistrer les modifications** lorsque vous modifiez le type de condition dans la section Détails du projet.
